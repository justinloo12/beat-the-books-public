Methodology & Honesty Notes

What this is

A fully automated MLB totals model, published daily and graded against real results. The point of this page is not to sell picks — it is to measure, in public, whether the model has any edge at all. The only metric that can establish that quickly is closing line value (CLV): whether the prices the model bets at are consistently better than the prices available at first pitch.

How the model works

Each game is scored by independent modules, blended with hand-set config weights:

  • Pitcher — Statcast expected stats, contact quality allowed, arsenal run values for both starters.
  • Offense / lineup — lineup-vs-arsenal matchups, handedness splits, confirmed vs projected lineups.
  • Bullpen — availability, recent usage, and quality of each relief corps.
  • Context — park factors, weather (wind, temperature, air density), and umpire tendencies.

The blended run projections drive a deterministic run-distribution engine that prices totals; edge is the model's probability minus the no-vig market probability from DraftKings quotes. Only mid-band edges (3–6%) become picks — repeated backtests showed the model's biggest disagreements with the market are model error, not market error.

A stacked meta-model (logistic regression over the raw module signals plus the market's no-vig probability) is wired into the pipeline but deliberately inert: it trains only once 150+ graded picks with stored module signals exist. Until then every number here comes from the hand-set blend, and the status card above says so.

How CLV is measured — and its limits

  • An intraday workflow snapshots DraftKings odds ~4×/day, timed just before the main MLB game windows.
  • For each pick, the last snapshot captured before that game's first pitch is used as the closing price.
  • CLV = (no-vig probability of the pick's side at close) − (no-vig probability at pick time). Positive means the market moved toward the pick.
  • This is a proxy, not the true close: one book, and up to a few hours before first pitch depending on snapshot timing. Totals CLV is only computed when the closing snapshot quotes the same line the pick was made at.
  • Early archive days had a single daily snapshot, so their measured CLV is 0.0 by construction — those entries prove nothing in either direction.

Read the numbers honestly

  • Anything under ~100 graded bets is noise. Every table on this page carries its sample size; below n=100 a coin-flipping bettor would frequently produce the same record, good or bad.
  • ROI on a few dozen bets can swing ±30 points on variance alone. CLV converges faster than ROI, which is why it is the centerpiece metric here.
  • The archive only contains days the pipeline ran and validated — failed days are absent, which is itself a mild survivorship bias.
  • Grading uses flat 1u stakes at the archived odds. No bankroll heroics, no cherry-picked windows.
